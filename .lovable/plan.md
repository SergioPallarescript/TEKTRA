

## Current State of Notifications

### Where notifications ARE currently sent:
1. **CFO Module** — When a document claim (reclamación) is sent, notifications go to agents with the allowed roles for that document.

### Where notifications SHOULD be sent but ARE NOT:
Based on the features and requirements discussed throughout this project:

2. **Planos Válidos** — When a DO or DEM uploads a new plan or new version, ALL project agents should be notified.
3. **Libro de Órdenes** — When a new order is created, relevant agents (CON, PRO) should be notified. Also when an order requires validation (AI-detected project change).
4. **Libro de Incidencias** — When a new incident is created, all project agents should be notified.
5. **Admin Panel** — When a user's role is changed or an agent is removed from the project.
6. **Invitations** — When a new agent is invited to the project (this is handled by email, but an in-app notification could also be useful once they accept).
7. **Gantt Diagram** — When milestones are regenerated by DO/DEM, other agents should be notified of the update.
8. **Order Edits/Deletions** — When an order or incident is edited or deleted, agents should be aware of the change.

## Plan: Implement Missing Notifications

### Step 1: Add notifications to Planos Válidos
- In `PlansModule.tsx`, after a successful plan/version upload, insert notifications for all project members.

### Step 2: Add notifications to Libro de Órdenes
- In `OrdersModule.tsx`, after creating a new order, notify all project members (except the creator).
- After editing or deleting an order, notify project members.

### Step 3: Add notifications to Libro de Incidencias
- In `IncidentsModule.tsx`, after creating a new incident, notify all project members (except the creator).

### Step 4: Add notifications to Admin Panel
- In `AdminPanel.tsx`, after role changes or agent removal, notify the affected user.

### Step 5: Add notifications to Gantt Module
- In `GanttModule.tsx`, after regenerating milestones, notify all project members.

### Technical approach
- Query `project_members` to get all `user_id` values for the project.
- Insert a notification for each member (excluding the actor).
- Reuse the existing notification panel and realtime subscription in `AppHeader.tsx`.
